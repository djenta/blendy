# Blendy 2.0.0

Blendy 2 turns the desktop chat into a closed-loop Blender coach.

## What is new

- A plain-language readiness check shows whether Blender, LM Studio, and a usable model are actually connected.
- Blendy detects the loaded model's vision, tool-use, and context capabilities instead of guessing.
- Gemma 4 receives a cleaner prompt, one copy of the current question, images in the correct order, and model-appropriate generation settings.
- Each answer becomes a current checkpoint with quick actions to check the work, get unstuck, or locate the exact Blender control.
- A Project Notebook lives with the chat. It remembers intent without forcing the chat to one `.blend` file.
- Reference images can be attached to a question for visual comparison.
- Named activity stages and a Stop control make slow local generation understandable and cancelable.
- Evidence receipts are now built from the scene facts and tools that actually ran.
- Web access has three clear modes: Local only, Ask me, and Automatic.
- The Blender bridge uses a rotating local capability token, and model-directed web reads are time, size, redirect, content-type, and private-network limited.
- The packaged Electron runtime is current, unused dependencies are removed, and GitHub now verifies every change and builds releases from version tags.

## Safety boundary

Blendy remains a tutor. It can inspect context and explain steps, but it does not run model-written Blender Python or silently change the scene.

## Install

Run `Blendy Local AI Tutor Setup 2.0.0.exe`. The installer updates the desktop app and copies the bundled add-on into detected Blender versions.

The installer is currently unsigned. Windows may show an unknown-publisher
warning. Compare the installer with `SHA256SUMS.txt` from this release if you
want to verify the GitHub-built file before running it.
