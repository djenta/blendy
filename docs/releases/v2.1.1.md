# Blendy 2.1.1

This patch improves Blendy's general modeling judgment and recovery behavior.

## What changed

- New-project guidance now chooses the starting primitive from the target's
  main silhouette and topology instead of treating Blender's default Cube as
  progress that must be preserved.
- Blendy prefers replacing a poor foundation when that is cleaner than adding
  repair topology, modifiers, or extra beginner steps.
- In-progress geometry is still reused when it genuinely suits the intended
  form; selection alone is no longer a reason to keep it.
- More natural correction language now invalidates the prior assistant answer,
  including reports that the visible result does not match the instructions.
- Correction and troubleshooting turns use the conservative exact sampling
  profile more reliably.
- New-project goal anchors are extracted dynamically from the user's stated
  project, so the same reasoning applies across unrelated models.
- Empty tool lists now report zero tool-definition tokens in the context meter.

## Validation

- 4 Node regression and mocked lifecycle suites passed.
- 82 Python bridge, security, context, screenshot, and version-alignment tests
  passed.
- The production TypeScript/Vite build and Windows installer completed
  successfully.

## Install

Run `Blendy Local AI Tutor Setup 2.1.1.exe`. The installer updates the desktop
app and its managed Blender add-on.
